Projet : Principes SOLID

Principes :
- **SRP (Single Responsibility Principle)** : Une classe ne doit avoir qu’une seule responsabilité.
- **OCP (Open/Closed Principle)** : Le code doit être ouvert à l’extension mais fermé à la modification.
- **LSP (Liskov Substitution Principle)** : Une sous-classe doit pouvoir remplacer sa classe mère sans erreur.
- **ISP (Interface Segregation Principle)** : Il vaut mieux plusieurs petites interfaces qu’une grande.
- **DIP (Dependency Inversion Principle)** : Les classes doivent dépendre des abstractions et non des implémentations.
